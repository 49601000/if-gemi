# conversation.html 仕様・マッピング

`conversation.html` は、比喩/構造化テキストを「ジェミ師匠と僕の会話」に変換するための**プロンプト生成ツール**です。  
このページの各UI値は、コード側で分岐して会話を決定するのではなく、主に `buildPrompt()` 内の指示文として埋め込まれ、LLMの出力傾向を調整します。

## Quick Mental Model
- 6パラメータは**決定論的分岐ではなく、LLMへの prompt steering**（指示圧調整）
- 同じ値でもモデル差・モデルバージョン差で効き方は多少変動する
- 最終品質は実出力を見ながら調整する前提
- 実際の出力挙動に効くルールは、必ず `conversation.html` の `buildPrompt()` 側へ反映する

### 非対象（今回の方針）
- `conversation.html` から `conversation-spec.md` を `fetch` して仕様プレビュー表示する機能は追加しない
- `conversation-spec.md` はドキュメント用途（仕様メモ）であり、ランタイム制御には使わない

### ノブ早見表
| ノブ | 変えるもの |
| --- | --- |
| 構造分解度 | 頭の良さ |
| 会話劇度 | エンタメ性 |
| 弟子理解遅延 | 読者代理度 |
| 師匠説明圧 | 教材感 |
| 比喩保持度 | 元ネタ依存度 |
| パンチライン | 締めの強さ |

## Detailed Mapping

### 6パラメータの反映方式
`buildPrompt()` では `X/10` だけでなく、`scaleLabels` と `getScaleLabel()` で意味ラベルを併記します。

出力形式:

```text
- 構造分解度：8/10（高め。論点の分解と対応関係を明確にする）
```

`getScaleLabel(id, value)` の段階:
- `0-1`: 最小
- `2-3`: 低め
- `4-6`: 標準
- `7-8`: 高め
- `9-10`: 最大

### 各段階のプロンプト定義マッピング
#### 構造分解度（`structureDepth`）
| 値域 | プロンプト上の定義 |
| --- | --- |
| 0-1 | 直感重視。構造説明は最小限にし、読者の感覚で伝える |
| 2-3 | 軽め。要所だけ論点を分解する |
| 4-6 | 標準。必要に応じてSource/Target対応を補助的に示す |
| 7-8 | 高め。論点の分解と対応関係を明確にする |
| 9-10 | 最大。構造対応と論点分解を明示して組み立てる |

#### 会話劇度（`dialogueDrama`）
| 値域 | プロンプト上の定義 |
| --- | --- |
| 0-1 | 解説寄り。会話は説明の受け皿として使う |
| 2-3 | 控えめ。自然な質疑応答を中心にする |
| 4-6 | 標準。師匠と弟子の掛け合いを自然に入れる |
| 7-8 | 高め。驚き・ツッコミ・感情の動きを強める |
| 9-10 | 最大。芝居がかった展開や間を使い、読み物としての会話劇を強める |

#### 弟子理解遅延（`studentDelay`）
| 値域 | プロンプト上の定義 |
| --- | --- |
| 0-1 | 即理解。弟子は早めに納得する |
| 2-3 | 軽め。少し疑問を挟む程度 |
| 4-6 | 標準。途中で誤解しながら段階的に理解する |
| 7-8 | 高め。何度かズレた反応をして、読者代理として引っ張る |
| 9-10 | 最大。終盤まで腑に落ちず、最後に理解する |

#### 師匠説明圧（`teacherDensity`）
| 値域 | プロンプト上の定義 |
| --- | --- |
| 0-1 | 短く刺す。師匠は説明をかなり削る |
| 2-3 | 軽め。短い補足で導く |
| 4-6 | 標準。必要な説明は入れるが長文化しない |
| 7-8 | 高め。理由や背景をやや丁寧に説明する |
| 9-10 | 最大。教材寄りに、論点をかなり丁寧に教える |

#### 比喩保持度（`metaphorRetention`）
| 値域 | プロンプト上の定義 |
| --- | --- |
| 0-1 | 自然使用。比喩は一度だけ軽く使う |
| 2-3 | 控えめ。要所で短く使う |
| 4-6 | 標準。比喩を理解補助として複数回使う |
| 7-8 | 高め。元比喩の要素をかなり残して説明する |
| 9-10 | 最大。元比喩の構造・感情・語彙を強く保持する |

#### パンチライン（`punchlineStrength`）
| 値域 | プロンプト上の定義 |
| --- | --- |
| 0-1 | 穏やか。余韻重視で締める |
| 2-3 | 軽め。軽いツッコミで終える |
| 4-6 | 標準。印象に残る一言で締める |
| 7-8 | 高め。読者に刺さる強めの反転や皮肉を入れる |
| 9-10 | 最大。最後に強い一撃のパンチラインで落とす |

### 思想・価値観の反映タイミング
`philosophyTiming` は次の4モード:
- `all`: 全体に反映
- `twist`: 終盤のどんでん返しで反映（デフォルト）
- `final`: 締めの一言だけに反映
- `none`: 反映しない

`thoughts` が空、または `none` の場合も、プロンプトに状態を明示して誤適用を防ぎます。

### ジェミ師匠のひとこと
`includeJemiHitokoto` が ON のとき:
- `【師匠と弟子の会話】` の後、`【構造メモ】` の前に挿入
- `jemiHitokotoCount`（1-5、既定3）を件数目安として要求
- 各項目は `テーマ名：コメント` の形式で出力
- テーマ名は本文の単語をそのまま使わず、概念ラベルへ要約（比喩の語を直貼りしない）
- コメントは**説明ではなく、読者へ直接入るツッコミ口調**で生成
- 長く説明せず、1〜2文の短いツッコミにする
- `〜とは` `〜ということ` `〜である` `定義上` `本質的に` など解説口調は禁止
- 単なる本文要約の焼き直しは禁止（教訓・皮肉・補足ツッコミのいずれか1軸で書く）

OFF のとき:
- `【ジェミ師匠のひとこと】は出力しない` を明示

#### 勝ちパターン例（参考）
- 思考停止の放置：それ慎重ちゃうで、怖くてブレーキ踏みっぱなしなだけやｗ
- 参加圧：みんなやっとるからって、お前の人生まで団体行動にせんでええｗ
- 別ルート設計：一本に賭けるより、自分向けの勝ち筋を先に作っとけｗ

#### 負けパターン例（非推奨）
- 先送りバイアス：意思決定を先延ばしにすることは機会損失の原因である。
- 参加圧への誤認：同調圧力と主体的選択の差異を理解すべきである。
- 資産形成の多様性：投資以外にも複数の手段が存在するということ。

#### 例文の扱い
- 上記の例文はテンプレートではなく、語調・構造・ツッコミの質感を示す参考例
- 例文の文言やキーフレーズをそのまま再利用しない

## Implementation Notes
### DSLレイヤ（口調と代名詞ルール）
`mentorSpeechDsl` + `conversationConstraints` をプロンプト前段に注入します。

主な制約:
- 師匠の一人称: 基本 `ワイ`、たまに `ワシ`
- 師匠の二人称: `お前` または `弟子` のみ
- 弟子の一人称: `僕`（不自然なら省略可）
- 弟子が師匠を呼ぶ語: `師匠` 固定
- 禁止代名詞: `あなた` `君` `きみ` `俺` `私` `自分` など
- 長文化・説明臭・概念ラベル化・セミナー調リアクションを抑制

### 出力フォーマット
標準:
1. `【師匠と弟子の会話】`
2. `【ジェミ師匠のひとこと】`（ON時のみ）
3. `【構造メモ】`

会話ラベル例（表示名）:
- `ジェミ師匠「…」`
- `僕「…」`

### 保存仕様（localStorage）
conversationページ固有キー:

| 保存対象 | localStorageキー | 中身 |
| --- | --- | --- |
| 6スライダーの保存済み設定 | `moshiJemi_conversation_ranges` | 6パラメータ値 |
| ページ全体状態 | `moshiJemi_conversation_state` | 入力欄・出力欄・思想設定・ひとこと設定・コピー状態など |

補足:
- 入力/変更時はデバウンス保存（`scheduleSaveConversationState`）
- 生成時は即保存
- 「設定を保存」ボタンは `ranges` を永続保存し、起動時既定値として再利用
