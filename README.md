# 比喩プロンプトビルダー3点セット 役割まとめ

## スレタイ
**【整理】`metaphor.html` → `scenario.html` → `conversation.html` の順で使うと、比喩を「理解しやすい会話台本」まで持っていける件**

## 本文（なんjレス風）

1. **`metaphor.html` の役割（比喩を作る・点検する）**  
   比喩生成の土台ページ。  
   元ネタ本文と比喩の目的を入れると、Gateチェック付きの「比喩生成プロンプト」を作れる。  
   つまりこういうことや: **比喩の品質管理と初期生成担当**。

2. **`scenario.html` の役割（比喩をシナリオ構造へ翻訳する）**  
   `metaphor.html`で作った比喩を入力して、  
   Source/Targetや構造分解を明示した「シナリオ骨子生成プロンプト」に変換する。  
   つまりこういうことや: **比喩を論点付きで物語構造に落とす担当**。

3. **`conversation.html` の役割（師匠弟子の会話プロンプトを最終生成する）**  
   比喩・構造化出力・元ネタをまとめて入力し、  
   師匠キャラ/弟子キャラ/各種スライダーで調整して「会話文生成プロンプト」を作る。  
   つまりこういうことや: **最終アウトプットを会話形式に仕上げる担当**。

4. **3つの関係（A vs B で言うと）**  
   - `metaphor.html`: 比喩の**発明と検査**  
   - `scenario.html`: 比喩の**構造翻訳**  
   - `conversation.html`: 構造の**会話化（読者向け表現）**

5. **実運用フロー（推奨）**  
   1. `metaphor.html` で比喩を作る  
   2. その結果を `scenario.html` に入れてシナリオ骨子を作る  
   3. さらに `conversation.html` で師匠弟子の会話台本に仕上げる

## なんj民の補足・ツッコミ

- 「ファイル名修正後は `scenario.html` / `conversation.html` で統一。読みやすさも保守性も上がるで」  
- 「役割が被ってるようで被ってない。前段の出力を次段で再利用する“パイプライン構成”や」
